## 工夫した点<br>
・更新ボタン　→　ラジオボタン出現（１つのみ選）<br>
　削除ボタン　→　チェックボックス出現（複数選択可能）<br>
　とし、間違えて操作されないように段階を踏むようにしました。<br>
　クリックの回数が増えユーザーとしては操作性に欠けるという欠点もありますが、<br>
　「安全性」を優先したため、この形にしました。<br>

・idは、メモ追加時に「1番大きい数+1」で自動付与しています。<br>
　そのため、削除されると、永久欠番になります。<br>
 
・ラジオボタン、チェックボックスを出現する時にメモのメモテーブルのidを取得し、<br>
　そのidをキーに、SQL文作成・実行しました。

・各項目で「並べ替え」機能をつけ、流れや塊として把握しやすいようにしました。<br>

・一つずつ機能を実装しながら、改善・試行錯誤しました。<br>

## 苦労や学び<br>
・Java（Servletやmodel）とJSPのそれぞれの役割の違いに、<br>
　最初は戸惑いました。<br>
　それぞれで保存したものが連携しているわけではなく、<br>
　取得したりインポートしてやっと使用できるということを<br>
　自分に叩き込むようにして失敗を繰り返し慣れていきました。<br>
　そのおかげで、処理と画面を分けて考える重要性を実感できました。<br>
　→この点を踏まえ、後日「メモアプリ」を制作。<br>

・DBやDAO、テーブル、SQLについての理解が難しく、<br>
　実際にたくさんプログラムを書いて、動かすことで、少しずつ理解が深まっていきました。<br>

・テーブル作成時に、「特定の人だけのメモ」というのを考慮していなかったため、<br>
　最終的に「掲示板アプリ」ということにしました。<br>
　→この点を踏まえ、後日「メモアプリ」を制作。<br>

・それぞれの関係性を考慮すると難しい部分もありましたが、<br>
　想像していたものが形になるという達成感と楽しさがありました。<br>
